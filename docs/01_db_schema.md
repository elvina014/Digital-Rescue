데이터베이스 스키마 설계 (Database Schema)1. 개요본 문서는 Digital-Rescue 프로젝트의 Supabase(PostgreSQL) 데이터베이스 구조를 정의한다.직원 권한(Role), 수리 접수건(Ticket), 부품 재고(Inventory), 모델별 견적(Device Model) 데이터를 효율적으로 관리하고 무결성을 유지하는 것을 목표로 한다.2. 공통 Enum (열거형) 타입2.1 EmployeeRole (직원 권한 등급)ADMIN: 관리자 (모든 권한)MANAGER: 팀장 (승인, 분쟁 조정, 전체 진행 관리)RECEPTION: 접수처 (신규 접수, 담당기사 배정)TECHNICIAN: 담당기사 (견적 산출, 고객 응대, 자재비 승인 요청)EXPERT_REPAIR: 정밀수리팀 (복잡한 수리, 외주 관리)CS: 고객 서비스 (사후 관리, 불편 접수)2.2 TicketStatus (접수건 처리 상태)NEW: 신규 접수 (담당자 미배정)ASSIGNED: 담당자 배정 완료 (1차 상담 대기)IN_PROGRESS: 수리/점검 진행 중WAITING_APPROVAL: 자재비 및 견적 회사 승인 대기COMPLETED: 수리 완료 (결제 완료 및 출고)CANCELED: 수리 취소 (반출)2.3 ReceiptType (접수 방식)VISIT: 방문 서비스DELIVERY: 퀵/택배 서비스WALK_IN: 내방 서비스2.4 InventoryCondition (재고 상태)NEW: 신품GOOD: 양품 (중고 A급)DEFECTIVE: 불량품SURPLUS: 잉여 부품(중고 B/C급)3. 핵심 테이블 명세 (Tables)3.1 employees (직원 테이블)Supabase Auth의 auth.users와 1:1로 매핑되는 프로필 테이블.|| 필드명 | 타입 | 제약 조건 | 설명 || id | UUID | PK, FK(auth.users.id) | 직원 고유 ID || name | VARCHAR | NOT NULL | 직원 이름 || role | EmployeeRole | NOT NULL | 직원 등급 (관리자, 팀장 등) || phone | VARCHAR |  | 연락처 || created_at | TIMESTAMPTZ | DEFAULT NOW() | 생성일 |3.2 customers (고객 테이블)| 필드명 | 타입 | 제약 조건 | 설명 || id | UUID | PK | 고객 고유 ID || name | VARCHAR | NOT NULL | 고객명 (필수) || phone | VARCHAR | NOT NULL | 휴대폰 번호 (필수) || address | VARCHAR |  | 주소 (방문/택배 시 필요) || created_at | TIMESTAMPTZ | DEFAULT NOW() | 생성일 |3.3 repair_tickets (수리 접수건 테이블) - (타임라인 역할)| 필드명 | 타입 | 제약 조건 | 설명 || id | UUID | PK | 로그 고유 ID || ticket_id | UUID | FK(repair_tickets.id) | 대상 접수건 ID || employee_id | UUID | FK(employees.id) | 작성 직원 ID || message | TEXT | NOT NULL | 처리 현황 내용 / 특이사항 || created_at | TIMESTAMPTZ | DEFAULT NOW() | 작성 일시 |4. 주요 비즈니스 로직 제약사항 (AI 구현 시 주의점)견적 산출 공식: final_price는 device_models 테이블을 참조하여 계산된 initial_estimate(감가상각 반영 최소 견적)보다 낮게 설정할 수 없도록 애플리케이션 단에서 검증(Validation)해야 함. 만약 낮다면 별도의 승인 로직이 필요.수정 불가 정책: repair_tickets.is_approved가 TRUE (회사 승인 완료)로 변경된 이후에는 ADMIN 권한을 제외한 어떤 직급도 해당 접수건의 금액이나 상태를 수정할 수 없다.RLS (Row Level Security): Supabase RLS를 적용하여, 대고객 홈페이지에서는 인증되지 않은 사용자가 repair_tickets 데이터를 무단 조회할 수 없게 막고, 로그인한 직원만 자신의 권한에 맞는 데이터를 조회/수정할 수 있도록 설정한다.
