데이터베이스 스키마 설계 (Database Schema)1. 개요본 문서는 Digital-Rescue 프로젝트의 Supabase(PostgreSQL) 데이터베이스 구조를 정의한다.직원 권한(Role), 수리 접수건(Ticket), 부품 재고(Inventory), 모델별 견적(Device Model) 데이터를 효율적으로 관리하고 무결성을 유지하는 것을 목표로 한다.2. 공통 Enum (열거형) 타입2.1 EmployeeRole (직원 권한 등급)ADMIN: 관리자 (모든 권한)MANAGER: 팀장 (승인, 분쟁 조정, 전체 진행 관리)RECEPTION: 접수처 (신규 접수, 담당기사 배정/변경)TECHNICIAN: 담당기사 (견적 산출, 고객 응대, 자재비 입력, 승인 요청)EXPERT_REPAIR: 정밀수리팀 (복잡한 수리, 외주 관리, 자재비 입력)CS: 고객 서비스 (사후 관리, 불편 접수)2.2 TicketStatus (접수건 처리 상태)NEW: 신규 접수 (담당자 미배정)ASSIGNED: 담당자 배정 완료 (1차 상담 대기)IN_PROGRESS: 수리/점검 진행 중WAITING_APPROVAL: 자재비 및 견적 회사 승인 대기COMPLETED: 수리 완료 (결제 완료 및 출고)CANCELED: 수리 취소 (반출)2.3 ReceiptType (접수 방식)VISIT: 방문 서비스DELIVERY: 퀵/택배 서비스WALK_IN: 내방 서비스2.4 InventoryCondition (재고 상태)NEW: 신품GOOD: 양품 (중고 A급)DEFECTIVE: 불량품SURPLUS: 잉여 부품(중고 B/C급)3. 핵심 테이블 명세 (Tables)3.1 employees (직원 테이블)Supabase Auth의 auth.users와 1:1로 매핑되는 프로필 테이블.필드명타입제약 조건설명idUUIDPK, FK(auth.users.id)직원 고유 IDnameVARCHARNOT NULL직원 이름roleEmployeeRoleNOT NULL직원 등급 (관리자, 팀장 등)phoneVARCHAR연락처created_atTIMESTAMPTZDEFAULT NOW()생성일3.2 customers (고객 테이블)필드명타입제약 조건설명idUUIDPK고객 고유 IDnameVARCHARNOT NULL고객명 (필수)phoneVARCHARNOT NULL휴대폰 번호 (필수)addressVARCHAR주소 (방문/택배 시 필요)created_atTIMESTAMPTZDEFAULT NOW()생성일3.3 repair_tickets (수리 접수건 테이블) - 가장 핵심필드명타입제약 조건설명idUUIDPK접수건 고유 IDcustomer_idUUIDFK(customers.id)접수 고객assignee_idUUIDFK(employees.id), NULL배정된 담당기사statusTicketStatusDEFAULT 'NEW'현재 처리 상태receipt_typeReceiptTypeNOT NULL접수 방식 (택배, 내방 등)device_brandVARCHARNOT NULL브랜드명 (예: samsung, msi)device_modelVARCHARNULL기기 모델명symptomsTEXTNOT NULL고장 증상 및 문의 내용initial_estimateINTEGERDEFAULT 0시스템 산출 최소 견적 (DB 기준)expected_estimateINTEGERDEFAULT 0예상 견적 (기사가 고객과 조율한 수리 예상 금액)material_costINTEGERDEFAULT 0자재비 총합 (수리에 사용된 부품 구매 총액)material_cost_detailsJSONBDEFAULT '[]'[추가] 자재비 상세 내역 (비용내용, 금액의 배열)final_priceINTEGERDEFAULT 0최종 확정 견적 (수리 종료 후 실제 결제할 금액)is_approvedBOOLEANDEFAULT FALSE회사(팀장/관리자) 최종 승인 여부payment_statusVARCHARDEFAULT 'PENDING'결제 상태 (PENDING, PAID)payment_methodVARCHARNULL[추가] 결제 방식 (CARD, BANK_TRANSFER, E_PAYMENT 등)created_atTIMESTAMPTZDEFAULT NOW()접수 일시updated_atTIMESTAMPTZDEFAULT NOW()마지막 수정 일시3.4 device_models (모델별 견적 DB 테이블)(생략 -
